# 项目部署指南 (GitHub + Vercel)

本文档将指导你如何将本项目推送到 GitHub，并配置 Vercel 进行自动化部署。

## 1. 准备工作

本项目已经配置了 `vercel.json`，用于支持单页应用 (SPA) 的路由重写。`.gitignore` 也已配置好，忽略了不必要的文件。

## 2. 推送到 GitHub

你需要在 GitHub 上创建一个新的空仓库（Repository）。
假设你创建的仓库地址为 `https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git`。

请在终端中执行以下命令（请替换 `URL` 为你实际的仓库地址）：

```bash
# 1. 暂存所有更改 (包括新添加的 vercel.json 和本文档)
git add .

# 2. 提交更改
git commit -m "docs(deploy): 添加 Vercel 配置文件和部署指南"

# 3. 关联远程仓库 (如果你还没有关联过)
# 注意：将下面的 URL 替换为你自己的 GitHub 仓库地址
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git

# 4. 推送到 GitHub
git push -u origin master
```

> **注意**：如果 `git remote add` 报错提示 "remote origin already exists"，说明已经关联过。你可以用 `git remote -v` 查看当前关联的地址。如果是旧地址，可以用 `git remote set-url origin NEW_URL` 修改。

## 3. 在 Vercel 上配置自动部署

1.  访问 [Vercel Dashboard](https://vercel.com/dashboard) 并登录。
2.  点击 **"Add New..."** -> **"Project"**。
3.  在 **"Import Git Repository"** 列表中找到你刚刚推送的 GitHub 仓库，点击 **"Import"**。
4.  **Configure Project**:
    *   **Framework Preset**: Vercel 通常会自动识别为 `Vite`。如果没识别，手动选择 `Vite`。
    *   **Root Directory**: 保持默认 (./)。
    *   **Build & Output Settings**: 应该已经被自动填充 (Build Command: `vite build`, Output Directory: `dist`)。
5.  点击 **"Deploy"**。

## 4. 验证部署

部署完成后，Vercel 会提供一个访问域名。点击链接，测试你的应用：
*   验证首页是否加载正常。
*   点击各个工具链接，验证路由跳转是否正常 (如果不正常，可能是 `vercel.json` 未生效或配置有误)。

## 5. 自动部署

完成上述配置后，每当你向 GitHub 仓库的 `master` 分支推送代码时，Vercel 会自动检测到变更并触发新的部署。你无需做任何额外操作。
